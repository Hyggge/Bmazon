# Bmazon 项目开发文档

## 一、项目背景

目前校内存在多种交易行为，例如二手闲置物品交易、社团班级等同统一物资采购、快递外卖代取等，然而这些交易体验并不好，有很多痛点：

- 二手交易有多个群，不同群之间存在信息差，可能A同学在B群发布了一个闲置物品，B群内没有同学购买，C同学不再B群中但是正好需要这个闲置物品；再或者某个物品在多个群售卖，价格相差较大。
- 社团班级统一采购物资普遍以聊天群中以统计的形式进行，很不方便，同学不能充分了解商品和评价，甚至购买时清楚价格，统计实名制给同学带来了心理压力等。
- 外卖代取局限在小范围内，不能一次性尽可能多的捎带，真正有需求的同学找不到代取渠道。

总而言之，校内的交易缺少一个统一、透明、可靠的平台。因此，笔者开发了该校内交易平台，以期帮助同学们在便捷有保障的情况下完成校内交易。



## 二、项目设计

为了解决上面这些“痛点”，我对项目进行了如下设计——

- 为解决多个群信息不透明问题，我们应该让每个同学都能在这个平台上发布自己的闲置二手物品，每个同学都能看到所有的在售商品。

- 交易平台应该可靠，不能出现下单了不发货这种情况。为此，在开店之前我们需要先进行学生认证，确认开店用户是在校学生才允许开店售卖商品，如果出了问题也可以直接通过这些信息找到相关学生。

- 为了解决统一采购相关问题，平台应该具有订单评价功能，要能看到商品详细信息，同时下单对于卖家应该是匿名的（不使用真实姓名购物）。

- 为了解决外卖代取等问题，可以设置虚拟商品——外卖、快递代取。

- 为了让同学种草推广商品，应该提供文章功能，同学可以写文章来为自己的商品“代言”

  

## 三、项目亮点

- 实体数量较多，实体之间有复杂的一对多、多对多关系。

- 平台功能丰富，囊括了电商用户大多数需求。

- 使用JWT进行用户身份验证，且用户密码都以密文的形式进行存储，极大的保证了安全性

- 前端使用element-ui组件库，动画效果丰富，界面简洁优雅，提高用户了体验。

- 项目工程量大，业务逻辑复杂。

  

## 四、项目相关技术

### (1) 前端

本项目的前端主要使用Vue.js框架进行搭建，技术栈包括Vue2，Element-UI，View-UI，Vuex，Webpack，Elint, Axios 等等。

#### 主要技术介绍

##### Vue.js

Vue.js是一套用于构建用户界面的渐进式js框架，其主要优势包括——

- 进行组件化开发,大大减少了代码编写量,便于理解和维护。
- 实现js数据和HTML元素的双向绑定，减少不必要的dom操作，大大降低了开发时的复杂度。
- 有着丰富多元的第三方插件资源，减少“重复造轮子”，开发起来更加高效便捷。
- vue开发出来的项目是单页面应用，每次跳转只需要进行局部组件的刷新，大大加快了页面访问速度，提高了用户体验。

##### Element-UI

Element-Ul是饿了么前端团队推出的一款基于Vue.js 2.0 的桌面端UI组件库，其中有很多已经封装好的Vue组件，包括输入框、文本框、按钮等等，可以满足绝大多数的开发需求。Element提供的组件不仅简单易用，而且设计简约优雅，即使没有很高超的css和js技术也能实现较好的页面效果。

##### Axios

Axios 是一个基于 Promise 的 HTTP 库，用于向后端服务器请求数据，便于前后端分离开发。Axios使用简单,包尺寸小且提供了易于扩展的接口，不仅支持Promise的所有API，还可以实现对请求和响应的拦截，便于对请求内容和响应数据进行预处理。

为了能统一处理一些常规事务，如在请求头中加入token、出现请求错误后向用户发送错误信息等等，笔者对Axios进行了二次封装，极大的提高了代码的可复用性，减少了维护成本。此外，二次封装也能便于我们更好地管理接口，使得在接口较多的情况下不至于出现混乱。

#### 主要模块

前端页面主要由以下6个模块组成：

- **登陆注册模块**：用户登陆与注册

- **用户模块**：用户个人信息、学生认证、个人订单、商品收藏

- **管理员模块**：用户列表展示、店铺列表展示、订单列表展示、学生认证列表展示

- **商品模块**：商品创建、全部商品展示、商品详情

- **店铺模块**：创建店铺、店铺详情、店铺商品、店铺订单

- **文章模块**：创建文章、修改文章、全部文章展示、文章详情

  

### (2) 后端

本项目的后端采用rails7框架，同时使用Sqlite作为数据库存储用户输入，使用Active Storage进行图片存取，使用jwt技术进行用户鉴权。

#### 主要技术介绍

##### Active Storage

Active Storage是实现图片上传一种安全简便的方法，被rails开发人员广泛使用。它不仅可以将用户文件上传到本地磁盘，还可以将上传到云存储服务（如 Amazon S3、Gogle Cloud Storage、Microsoft Azure Storage）。更重要的是，它能够将文件附加到 Active Record 对象，开发者可以直接通过Actice Record来操作文件，十分方便。本项目就是利用了这一点实现了图片（商品预览图、用户头像等）的上传和url获取。

##### JWT

 JSON Web Token（JWT）是一个开放的行业标准（RFC 7519），它定义了一种简洁的、自包含的协议格式，用于在通信双方传递json对象，传递的信息经过数字签名可以被验证和信任。JWT的主要特点是——安全、稳定、易用、支持JSON，这也使其成为目前最流行的跨域身份验证解决方案。在本项目中，笔者借用了'jwt'这一gem包实现了用户口令的加密和token的生成与解析。

## API文档

### (1) 登录注册相关API

#### 1. 登录

##### `[POST] /api/auth/login`

##### Body

* 必要字段：`username, passowrd`

  ```json
  {
      "username": "czh",
      "password": "123456"
  }
  ```

##### Response

* 成功则返回：``id`（用户id）`token`（JWT token）, `ddl`（token有效截止时间），`role`（0表示管理员，1表示非管理员）

  ```json
  {
      "id": 1,
  	"token": "eyJhbGciOiJIUzI1NiJ9.eyJ1c2VyX2lkIjoxLCJleHAiOjE2NzI4MTg3MDd9.SCkAmwUYgQYTuTg8oN0XOavRTIMYK8F7p6ha_gJwvDo",
  	"ddl": "01-04-2023 15:51",
  	"role": 0
  }
  ```



#### 2. 注册

##### `[POST] /api/auth/register`

##### Body

* 必要字段：`username, passowrd, email`

* 可选字段：`signature, phone_no`

  ```json
  {
      "username": "mlh",
      "password": "123456",
      "email": "mlhsdmv@qq.com",
  }
  ```

##### Response

* 成功则返回提示信息和用户`id`

  ```json
  {
  	"success": true,
  	"id": 3
  }
  ```

* 如果用户名已存在，会返回错误信息（由rails自动生成）

  ```json
  {
  	"error": {
  		"username": [
  			"has already been taken"
  		]
  	}
  }
  ```



#### 3. 检查用户名是否存在

##### `[GET] /api/auth/check_dup_username/<str:username>`

##### Param

* username 为要检查的用户名

##### Response

* 响应内容：exist 表示用户名是否存在

  ```json
  {
  	"exist": true
  }
  ```

### (3) 学生相关API

#### 1. 学生认证

##### `[POST] /api/students`

##### Body

- 必要字段：`student_id, student_name, depart, attendance_year, gender, image_id`

  ```json
  {
      "student_id": 20373379,
      "student_name": "陈正昊",
      "depart": 6,
      "attendance_year": 2022,
      "gender": 0,
      "image_id": 1
  }
  ```

##### Response

- 成功则返回提示信息和学生id

  ```
  {
  	"success": true,
  	"id": 20373379
  }
  ```

- 如果学号被认证过，则返回错误信息

  ```json
  {
  	"error": "the student id has been authenticated"
  }
  ```



#### 2. 检查学号是否已经认证过

##### `[GET] /api/students/:student_id/check_dup_student_id`

##### Param

- student_id 为要检查的学号

##### Response

- 返回字段：exist表示学号是否被认证过

  ```json
  {
  	"exist": true
  ```
